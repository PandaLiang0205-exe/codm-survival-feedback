## 1. 這個網站是什麼

一個給 CoDM「生存模式」玩家使用的意見平台，任何人不用註冊帳號就可以：

- **回報遊戲 BUG**：把你遇到的錯誤傳上來讓大家看到
- **提交遊戲建議**：對玩法、平衡、系統提出改進意見
- **投票支持別人的建議**：用「支持 / 不支持」表達你的意見
- **檢視已解決的 BUG**：管理員標記為已修復的 BUG 也可以查詢

所有送出的內容需要經過**管理員審核**才會公開顯示，避免灌水與不相關內容。

打開網站後你會看到：

- **右上角統計條**：

- **兩個主分頁**：`BUG 回報` 與 `遊戲建議`
- **每個分頁內**：新增按鈕、搜尋欄、排序下拉、內容列表

## 3. 玩家操作

### 3.1 切換語言

- 右上角 **`中文 / EN`** 按鈕，一鍵切換
- 語言選擇會記住，下次進來自動用上次的設定

### 3.2 瀏覽 BUG 回報

1. 點頂部 **`BUG 回報`** 分頁
2. 分頁下方會看到兩個切換：
   - **未解決**（預設）— 尚未修復、待官方處理的 BUG
   - **已解決** — 管理員標記為已修復的 BUG（會有綠色 `RESOLVED` 標籤、卡片淡化、標題劃線）
3. 點任一張卡片可進入 **BUG 詳情頁**，看完整說明與附上的圖片/影片
4. 詳情頁左上有 **`← 返回`** 回到列表

### 3.3 瀏覽遊戲建議

1. 點頂部 **`遊戲建議`** 分頁
2. 每張建議卡片下方有 **`▲ 支持`** 與 **`▼ 不支持`** 兩個按鈕，可看到目前票數
3. 點卡片進入 **建議詳情頁**，詳情頁底部也有同一組投票按鈕

### 3.4 投票（支持 / 不支持）

- 每個裝置（瀏覽器）對每一則建議可以投一次票
- 已投的方向會用**實心底色**標示（支持 = 綠色實心 / 不支持 = 紅色實心）
- **點擊行為**：
  - 沒投過 → 點擊：+1 該方向
  - **已投該方向 → 再點一次：取消投票（收回）**
  - **已投 → 點另一方向：切換方向**（原方向 -1、新方向 +1）
- 投票紀錄存在你自己的瀏覽器裡（localStorage），清除瀏覽器資料後會重置

### 3.5 搜尋與排序

**搜尋**：
- 每個分頁的頂端都有 **`搜尋關鍵字...`** 輸入框
- 輸入時即時篩選（不用按 Enter），會同時搜尋標題與內文
- 大小寫不敏感

**排序**：
- 右邊的 **`排序:`** 下拉選單可切換：
  - **最新優先**（預設）
  - **最舊優先**
  - **支持度最高**（僅遊戲建議分頁有）— 依「支持 − 不支持」的差值排序

### 3.6 新增 BUG 回報 / 遊戲建議

1. 進入對應分頁後，點 **`+ 新增回報`** 或 **`+ 新增建議`**
2. 填寫：
   - **標題**（必填，最多 200 字）
   - **說明**（必填，最多 4000 字）
   - **媒體（選填）**：可上傳圖片或影片，單檔上限 **25 MB**
3. 點 **`送出`**
4. 送出後畫面會顯示：`✓ 已送出，等待管理員審核。`
5. **審核通過前**你的回報**不會**出現在公開列表上；被管理員拒絕則會直接刪除
6. 想再送一筆請按 **`← 返回`** 回列表後再點新增

> ⚠ 上傳過大的檔案（超過 25 MB）或非圖片/影片格式會被前端擋下，會顯示紅色錯誤訊息。

## 4. 管理員操作

管理員登入後可以審核回報、編輯內容、刪除項目、標記 BUG 為已解決。

### 4.1 登入

1. 點頂部 **`管理`** 進入管理頁
2. 輸入管理員 Email + 密碼（由系統管理員在 Supabase 後台建立）
3. 點 **`登入`**

> 登入狀態會保存在瀏覽器裡，不用每次都登。登出前都算已登入。

### 4.2 待審核佇列（Pending）

登入後預設進入 **`待審核`** 分頁：

- 一次顯示所有**待審核**的 BUG 與建議（混排）
- **舊 → 新**排序（先進先審）
- 每張卡直接展開完整內容（標題、說明、媒體、票數）
- 每張卡右下角兩顆按鈕：
  - **綠色 `核准`** — 把該筆設為已核准，立刻在公開列表出現
  - **紅色 `拒絕`** — 直接**刪除**該筆（不保留）

### 4.3 已核准清單（Approved）

點頂部的 **`已核准`** 分頁切換：

- 顯示所有**已公開**的 BUG 與建議
- **新 → 舊**排序（較新的優先看到）
- 每張卡右下角按鈕：
  - **綠色 `標記為已解決`**（僅 BUG，若尚未解決時顯示）
  - **黃色 `取消解決`**（僅 BUG，若已解決時顯示）
  - **黃色 `編輯`**
  - **紅色 `刪除`**

### 4.4 編輯他人回報

1. 在 **`已核准`** 分頁找到要改的項目
2. 點 **`編輯`**，卡片會就地展開成表單
3. 修改標題或說明後點 **`儲存`**
4. 想放棄修改點 **`取消`**

> 目前不支援編輯媒體檔案（圖片/影片），只能改文字。

### 4.5 標記 BUG 為已解決

當某個 BUG 已經被官方修復：

1. 到 **`已核准`** 分頁
2. 找到該筆 BUG，點 **`標記為已解決`**
3. 該 BUG 會：
   - 從主頁「未解決」列表消失
   - 出現在主頁「已解決」列表
   - 卡片顯示綠色 `RESOLVED` 標籤，並半透明淡化、標題劃線
4. 若標記錯誤，可以點 **`取消解決`** 把它退回「未解決」狀態

### 4.6 刪除項目

- 在 **`已核准`** 或 **`待審核`** 分頁點紅色 **`刪除`** / **`拒絕`**
- 已核准項目按 **`刪除`** 會跳出**確認對話框**，再次確認才會真的刪除
- 待審核項目按 **`拒絕`** 直接刪除（沒有確認對話框）
- **刪除是永久性的，無法復原**，請小心操作

### 4.7 登出

點右上角 **`登出`**，會清除登入狀態並回到登入頁。

## 5. 常見問題

**Q：我送出的回報怎麼還沒出現？**
A：需要等管理員審核。如果太久沒出現可能是被拒絕（拒絕即刪除），可以重新送一次並讓內容更清楚。

**Q：我投錯票了怎麼辦？**
A：再點同一顆按鈕可以**收回**投票；或直接點另一顆按鈕**切換方向**。

**Q：為什麼投票只算一次？**
A：每個瀏覽器（每個裝置）對同一則建議只能投一次，避免灌票。換裝置或清除瀏覽器資料就能重投。

**Q：進網站時整個畫面黑黑的？**
A：可能是網路載入外部資源失敗，請按 **`Ctrl + F5`** 強制重新整理。如果還是不行請等 1 分鐘再試（外部 CDN 可能暫時不通），或按 F12 打開 Console 看有沒有錯誤訊息。

**Q：右上角瀏覽次數為什麼不動？**
A：同一個瀏覽器分頁的 session 內只算一次瀏覽（避免灌計數）。關掉分頁再開才會 +1。

**Q：檔案上傳失敗？**
A：確認檔案：① 沒超過 25 MB ② 是圖片或影片（不接受 zip、pdf 等）。

**Q：手機也能用嗎？**
A：可以，介面響應式，在手機上會自動縮排適應。
